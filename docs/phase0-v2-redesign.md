# VoiceBridge v2 — Phase 0 重新规划

> **问题：** v1 的会话创建+双人加入流程太复杂
> **目标：** 打开即用，单人操作，零配置

---

## 1. 用户是谁、解决什么问题

**用户：** 老大（TikTok 公会运营者），管理 10,000+ 拉美/中东主播

**场景：** 跟外国主播打 WhatsApp/视频通话时，需要实时翻译
- 老大说中文 → 主播听到西班牙语/阿拉伯语/英语
- 主播说外语 → 老大听到中文

**核心约束：** 老大一个人操作，主播不需要碰设备。翻译工具充当"同传耳机"。

---

## 3. v2 设计方案：单人翻译器

### 页面结构（唯一页面）

```
┌──────────────────────────────────┐
│  VoiceBridge 实时翻译             │
├──────────────────────────────────┤
│                                  │
│  我说：[中文 ▼]                   │
│  对方说：[西班牙语 ▼]              │
│                                  │
│  ┌────────────────────────────┐  │
│  │         🎤 开始翻译          │  │
│  └────────────────────────────┘  │
│                                  │
│  状态：● 正在听...                │
│                                  │
│  ┌────────────────────────────┐  │
│  │ 原文：你好，欢迎加入公会       │  │
│  │ 译文：Hola, bienvenido...   │  │
│  └────────────────────────────┘  │
│                                  │
│  翻译历史：                       │
│  你好 → Hola, bienvenido al...   │
│  ¿Cuánto pagan? → 你们付多少...   │
└──────────────────────────────────┘
```

### 操作流程

1. 打开页面 → 选择「我说」语言和「对方说」语言（默认中文↔西班牙语）
2. 点 **🎤 开始翻译**
3. 对着麦克风说话 → 自动识别 → 翻译 → 通过扬声器播放译文
4. 对方说话 → 麦克风收音 → 识别 → 翻译 → 通过耳机播放中文
5. 通话结束 → 关闭页面

**全程不需要创建会话、不需要分享链接、不需要第二台设备。**

## 5. 技术方案

### 架构

```
浏览器 (Web Audio API)
    ↕ 单条 WebSocket
FastAPI /ws/translate
    → Deepgram ASR（识别原声）
    → DeepSeek（翻译）
    → ElevenLabs TTS（合成语音）
    → 返回音频 → 浏览器播放
```

### 关键改动

| 文件 | 改动 |
|------|------|
| `app/main.py` | 删除 session 路由，新增 `/ws/translate` |
| `app/session.py` | 删除 |
| `app/ws.py` | 简化为单连接管线 |
| `static/index.html` | 重写为单人 UI |
| `static/session.html` | 删除 |
| `static/app.js` | 重写 |

### WebSocket 消息协议（简化）

```json
// 浏览器 → 服务器（音频数据）
{"type": "audio", "data": "<base64>"}

// 服务器 → 浏览器（翻译结果+音频）
{"type": "result", "original": "你好", "translated": "Hola", "audio": "<base64>"}
```
