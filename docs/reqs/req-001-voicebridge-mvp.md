# REQ-001：VoiceBridge 实时双向语音翻译系统

> **日期：** 2026-06-01  
> **角色：** PM  
> **状态：** ✅ 待老大审批  
> **依赖：** Phase 0 发现文档（GO）

---

## 1. 背景

老大需要一个实时双工语音翻译工具：用户说中文，对方实时听到英文；对方说英文，用户实时听到中文。双方听到的语音保留各自原始音色。用于跨国商务通话、公会主播沟通等场景。

## 2. 目标

- 用户在浏览器中打开页面 → 选择语言方向 → 开始对话
- 端到端翻译延迟 < 1.5 秒
- 翻译准确率 > 90%（日常对话场景）
- 双方音色可辨识（音色克隆）

## 3. MVP 功能（V1）

### 3.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|:--:|
| **实时音频采集** | 浏览器 Web Audio API 采集麦克风 | P0 |
| **VAD 语音活动检测** | 自动检测说话/静音，切分语音段 | P0 |
| **ASR 语音识别** | Deepgram 流式中/英文识别 | P0 |
| **实时翻译** | GPT-4o-mini 中↔英双向翻译 | P0 |
| **TTS 语音合成** | ElevenLabs 流式语音合成 | P0 |
| **音频播放** | 浏览器 Web Audio API 播放翻译后语音 | P0 |
| **WebSocket 双工** | 前后端全双工音频流通信 | P0 |
| **会话管理** | 创建/加入会话，支持分享链接 | P1 |
| **音色录制** | 首次使用时录制声音样本 | P1 |
| **音色克隆** | ElevenLabs Voice Clone 保留原始音色 | P1 |

### 3.2 用户流程

```
1. 用户A打开页面 → 录制声音样本(30s) → 选择"我说中文"
2. 生成分享链接 → 发送给用户B
3. 用户B打开链接 → 录制声音样本(30s) → 自动配为"我说英文"
4. 双方开始对话
5. A说中文 → B听到英文(A的音色)
6. B说英文 → A听到中文(B的音色)
```

### 3.3 页面结构

- **主页：** 创建/加入会话
- **会话页：** 对话界面（波形/状态指示/VAD指示灯）
- **设置页：** 音色录制、语言偏好、音量控制

### 3.4 不做（V1 范围外）

- 移动端原生 App（V2）
- 多于2人的群组翻译（V2）
- 其他语言对（V2：西语/阿拉伯语等）
- 离线模式
- 通话录音/回放
- 噪声抑制（依赖浏览器内置）

## 4. 验收标准

| # | 标准 | 测量方式 |
|---|------|---------|
| 1 | 端到端延迟 < 1.5s | 从说话→听到翻译的时间 |
| 2 | 翻译准确率 > 90% | 随机100句日常对话人工评估 |
| 3 | 音色可辨识 | 盲测：听翻译后语音能认出谁在说话 |
| 4 | Chrome 浏览器正常运行 | 手动测试 |
| 5 | WebSocket 重连机制 | 断网5秒内自动重连 |
| 6 | 两个用户可同时说话不冲突 | 双工测试 |

## 5. 技术约束

| 约束 | 说明 |
|------|------|
| 前端 | 纯 HTML/JS，无框架依赖（V1）；Web Audio API |
| 后端 | Python FastAPI + WebSocket |
| ASR | Deepgram 流式 API |
| 翻译 | GPT-4o-mini / DeepSeek（备选） |
| TTS | ElevenLabs Turbo v2.5 |
| 音色克隆 | ElevenLabs Voice Clone |
| 部署 | Docker + 单容器；域名 voicebridge.apptopup.io |
